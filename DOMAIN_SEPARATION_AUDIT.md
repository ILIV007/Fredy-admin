# Fredy v11.17.0 — Scheduler Domain Separation Audit Report

## Executive Summary

The scheduler Domain Model has been fully separated. Window fields (`time`, `windowEnd`) drive ALL scheduling decisions. Time-based fields (`scheduledTime`, `epochMs`, `publishedAt`) are display-only — used for UI, analytics, and ordering, but NEVER for publish decisions.

## Domain Model

### SCHEDULING FIELDS (used by scheduler to make publish decisions):
- `time` (windowStart) — "HH:MM" string
- `windowEnd` — "HH:MM" string

### DISPLAY-ONLY FIELDS (for UI/analytics, NOT for scheduling):
- `scheduledTime` — "HH:MM" string (random within window, for display)
- `epochMs` — number (window start epoch, for ordering only)
- `publishedAt` — number (actual publish time, set after publish)

## Publish Decision Audit

### Publish Decision Source: **Window Only**

The scheduler's `findDueSlot()` method uses ONLY:
1. `slot.time` (parsed as minutes-since-midnight) — compared with `nowMinutes`
2. `slot.windowEnd` (parsed as minutes-since-midnight) — for expiry check
3. `post.status` — "pending" / "published" / "failed" / "publishing"

No `epochMs`, `scheduledTime`, or any exact timestamp is used for publish decisions.

## epochMs Scheduling Usages: 0

All remaining `epochMs` references are:
- **Data passthrough** (carrying the value from plan to display): lines 158, 1199, 1248 in scheduler-service, line 148 in strategy-engine
- **Comments** documenting the architecture: lines 241, 253 in scheduler-service
- **Ordering** (sorting slots by window start): not used in scheduling decisions
- **Display** (shown in dashboard API response): manager.ts

## scheduledTime Scheduling Usages: 0

`scheduledTime` is a new display-only field. It is:
- Generated by TimeGenerator (random within window)
- Passed through strategy-engine to the plan
- Displayed in the dashboard API response
- NEVER used in any scheduling decision

## Window Scheduling References: ✓

- `findDueSlot()`: `nowMinutes < startMin - 10` (cron tolerance)
- `findDueSlot()`: `nowMinutes >= expiryMin` (window expiry, exclusive end)
- `status()`: `nowMinutes < startMin - 10` (pending filtering)
- `manager.ts`: `nowMinutes < startMin - 10` (due/pending calculation)
- `daily-planner.ts`: `nowMinutes >= startMin` (next slot)
- `strategy-engine.ts`: `timeStringToEpoch(date, post.time, tz)` (quiet hours)
